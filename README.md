# 圣诞钩索冒险 - 实验版游戏设计文档

## 概述

**目标**：  
在3天内完成一个**实验性原型**，重点是实现钩索核心机制，同时尝试不同的工具和方法，为后续扩展提供技术验证与设计方向。

**Yifan的角色**：负责核心技术实现，包括钩索物理系统和游戏机制的开发。  
**Yufei的角色**：负责尝试不同工具（AI美术、音乐生成、关卡设计工具等）并围绕钩索机制做实验。

---

## 核心任务与优先级

### 1. 核心机制 - 钩索实现
**目标**：完成Candy Cane钩索的核心功能，包括：
- **发射**：朝鼠标方向或指定方向发射钩索。
- **连接**：在满足条件时，钩索勾住挂点（挂点对象需有特定标记）。
- **摆动与断开**：角色挂在钩索上，可摆动以积蓄动量，玩家随时可断开钩索。

**实现要求**：
- **发射逻辑**：使用`Raycast`检测挂点，成功连接后生成一条可视化钩索（用`LineRenderer`）。  
- **挂点条件**：挂点必须属于特定Layer或Tag。  
- **摆动物理**：通过`DistanceJoint2D`或`HingeJoint2D`模拟连接效果，调试角色的质量、重力和阻力以获得流畅手感。  
- **断开时动量保留**：角色在断开钩索后保留当前速度和方向。

**可用资源**：  
- 你可以实现钩索的底层物理逻辑和参数调优。  
- 她可以尝试使用AI工具生成挂点对象的美术素材（如圣诞花环、灯饰）。

---

### 2. 关卡实验 - 挂点与地形布局
**目标**：创建简单的测试关卡，验证钩索机制的手感和关卡设计的可行性。

**实验任务**：
- **工具尝试**：使用Unity Tilemap或简单的Prefab工具搭建地形（平台、挂点）。
- **关卡布局**：  
  - 起点与终点之间有若干挂点和平台。
  - 包括一些特殊地形，如：滑冰路面（低摩擦）、动态挂点（缓慢移动）。  
- **反馈收集**：让玩家测试钩索摆动的流畅性和挑战性，并调整挂点间距和高度。

**优先级**：  
- 优先搭建水平或简单垂直布局，用于验证基本机制。
- 后期可加入更多元素（如风力影响挂点或易碎平台）。

---

### 3. AI美术与音效实验
**目标**：尝试生成适合圣诞主题的美术与音效素材，用于增强氛围或测试工具潜力。

**实验任务**：
- **美术**：
  - 使用AI工具生成圣诞场景挂点（如灯饰、雪花、Candy Cane样式）。
  - 生成简易的背景图（如雪景或圣诞村庄）。
- **音效**：
  - 使用AI生成背景音乐（轻快的圣诞旋律）。
  - 添加关键事件音效（如钩索成功连接、角色落地时的声音）。

**优先级**：非核心任务，作为增加趣味和氛围的补充。

---

### 4. 技术实验与工具尝试
**目标**：为后续开发探索潜力工具或优化方案。

**实验方向**：
- **钩索的改进与拓展**：  
  - 她可以尝试将`DistanceJoint2D`替换为自定义的物理连接方案，验证可控性。  
  - 你可以提供如何实现动态调节绳索长度或摆动力矩的技术建议。
- **关卡生成工具**：  
  - 她可以尝试在Unity内置的关卡工具之外，探索其他外部工具（如ProBuilder或TileMap插件）。
- **AI驱动的内容生成**：探索是否能快速生成大量适配的装饰性挂点。

## 技术支持与需求定义

### 钩索核心功能
**角色基础属性**：
- 使用`Rigidbody2D`，调整`Mass`和`Drag`，确保跳跃和摆动手感流畅。
- 通过`Animator`或简单的脚本，赋予角色基础移动和跳跃。

**发射逻辑**：
- 鼠标点击后，角色发射一条Candy Cane钩索（用`LineRenderer`表示）。
- 使用`Raycast`检测目标挂点，如果命中正确对象，连接成功。

**摆动与断开**：
- 利用`DistanceJoint2D`连接角色和挂点，并设置合适的摆动参数。
- 玩家按键时断开连接，并将当前动量保留到角色刚体中。

### 关卡搭建
**测试关卡要求**：
- 包含3-5个挂点，适合基础摆动实验。
- 添加一些简单的平台或障碍，测试动量积累和跳跃距离。

## 故事线设定

### 背景故事
圣诞老人的小精灵学徒不小心把礼物都散落在了圣诞村的各个角落。作为一名见习小精灵，你需要用神奇的拐杖糖（Candy Cane）在圣诞夜前收集所有礼物。这根魔法拐杖糖不仅可以帮助你在雪夜中穿梭，还能带给这个小镇一些特别的圣诞魔法。

### 叙事元素
- **场景氛围**：温暖的圣诞小镇，飘着雪花，到处都是彩灯和装饰
- **收集要素**：散落的礼物、圣诞装饰、丢失的驯鹿铃铛等
- **互动对象**：友好的雪人、圣诞树、温馨的小屋窗户

## gameplay拓展构想

### Candy Cane变种系统
不同口味的拐杖糖具有独特的物理特性：

1. **薄荷味 Candy Cane**
   - 特性：冰冷清凉
   - 效果：
     - 接触物体会结冰，创造临时立足点
     - 可以冷却过热的平台
     - 在空中留下冰晶轨迹，短时间内可以作为踏板

2. **辣味 Candy Cane**
   - 特性：火热激情
   - 效果：
     - 可以融化冰块障碍
     - 接触某些特定平台会使其暂时变软或膨胀
     - 摆动时产生火花效果，提供额外推力

3. **彩虹味 Candy Cane**
   - 特性：轻盈梦幻
   - 效果：
     - 钩索长度可以动态伸缩
     - 连接时产生彩虹轨迹，提供短暂的额外跳跃平台

### 环境互动机制
- **温度系统**：
  - 某些区域会随着玩家使用不同口味的Candy Cane而改变状态
  - 环境可以从结冰变为融化，或者反之
  - 创造动态的路径和解谜机会

- **组合效果**：
  - 不同口味的Candy Cane可以在特定条件下组合使用
  - 例如：用辣味融化冰块后，立即用薄荷味制造新的冰面造路

### 后期开发方向
1. **解谜要素**：设计需要合理运用不同Candy Cane特性的关卡
2. **收集系统**：通过收集特定物品解锁新的Candy Cane口味
3. **环境叙事**：通过场景细节讲述圣诞小镇的故事
4. **视觉反馈**：为不同口味的Candy Cane设计独特的视觉效果和粒子系统

## 音乐与美术优先级需求

### 核心美术资源（最高优先级）
1. **钩索视觉效果**
   - Candy Cane钩索的基础外观（简单的红白条纹效果）
   - 钩索发射和连接时的视觉反馈（简单的闪光或轨迹特效）
   - 钩索断开时的视觉提示（可以是简单的粒子效果）

2. **必要场景元素**
   - 基础挂点（圣诞花环或装饰灯）- 需要清晰可识别
   - 简单的平台（雪地、冰面等）
   - 玩家角色的基础外观（小精灵的简单造型）

### 核心音效（高优先级）
1. **gameplay音效**
   - 钩索发射音效（清脆的"咻"声）
   - 钩索连接成功音效（铃铛声或清脆的连接声）
   - 钩索断开音效（简单的弹性断裂声）
   - 角色移动和着地音效（轻盈的脚步声）

2. **环境音效**
   - 简单的风声背景音
   - 基础的雪地脚步声

### 次要美术资源（中等优先级）
1. **环境装饰**
   - 简单的雪花粒子效果
   - 基础的圣诞彩灯
   - 远景背景（简单的圣诞小镇剪影）

2. **UI元素**
   - 简单的开始界面
   - 基础的游戏内提示图标

### 音乐需求（中等优先级）
1. **背景音乐**
   - 主要风格：轻快、温馨的圣诞风格
   - 长度：30-60秒循环
   - 音量：不影响游戏音效的清晰度
   - 参考：简化版的圣诞颂歌编曲

### 技术要求
1. **美术规格**
   - 所有sprite资源保持像素对齐
   - 使用统一的像素密度
   - 保持文件大小合理（单个贴图不超过1024x1024）
   - 导出格式：PNG（带透明通道）

2. **音频规格**
   - 音效格式：WAV或OGG
   - 采样率：44.1kHz
   - 音效时长：不超过2秒
   - 背景音乐格式：OGG（循环无缝）

### 工具建议
- **美术生成**：
  - Midjourney用于生成基础贴图参考
  - Stable Diffusion用于生成装饰元素
  - Aseprite用于像素风格修改

- **音效制作**：
  - SFXR用于生成基础音效
  - Mubert用于生成背景音乐
  - Audacity用于音频编辑

注意：优先确保核心gameplay相关的视觉和音频反馈，其他装饰性内容可以在基础功能实现后逐步添加。
